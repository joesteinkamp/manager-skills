# Weekly Digest Template

Use this as the default response structure for `team-progress-update` in `digest` mode. Optimized for Slack, email, or a Notion page the team and adjacent partners read.

## Window

- **Team:**
- **Window:** [Start date] – [End date]
- **Notion scanned:** (yes / no — note source fallback if no)
- **Prior digest:** [date, link if applicable]

## Shipped This Week

For each item: who shipped it, what it is, link.

- **[Designer]** — [Item title]. [One-line plain description of what shipped and why it matters.] → [link]
- **[Designer]** — [Item title]. [One-line description.] → [link]

## In Flight

What's actively moving, with the designer driving it and the next milestone.

- **[Designer]** — [Item title]. Next milestone: [milestone, date]. → [link]
- **[Designer]** — [Item title]. Next milestone: [milestone, date]. → [link]

## Recognition

Specific, named, tied to specific work. No generic "great work everyone."

- 🎉 **[Designer]** for [specific work] — [why it's worth recognizing: scope, quality, parallel workstream juggling, cross-functional save, etc.].
- 🎉 **[Designer]** for [specific work] — [why].

## Risks & Asks

Lightweight — surface what could derail next week and what you need from partners. Detailed strategic asks belong in an exec readout, not here.

- ⚠️ **[Item / project]** is [status] because [root cause]. [Owner] is [action]. → [link]
- 🙏 We need [specific input/decision] from [team/person] by [date].

## Coming Up

Milestones, launches, or notable events in the next 7-14 days.

- [Date] — [Designer] — [Milestone / launch / event]
