# Platform Catalog

The platforms where designers maintain a professional presence, grouped by
category. For each platform: the URL pattern, how to find a designer there, and
notes on disambiguation. Work through the categories that match the candidate's
discipline; never skip the staples (portfolio site, Dribbble, Behance, Medium,
X, GitHub).

Handle reuse is the single most useful signal in this catalog. Once you know a
designer's handle on one platform, check that exact handle on every other
platform — `dribbble.com/janedraws`, `x.com/janedraws`, `github.com/janedraws`,
`medium.com/@janedraws` are often the same person.

## Portfolio & Personal Sites

The portfolio is the highest-value link. Designers host it many ways.

- **Custom domain.** Most common patterns: `firstnamelastname.com`,
  `firstlastname.com`, `firstname.design`, `firstnamelast.co`,
  `firstname.studio`, `firstnamexyz.com`. Try the obvious permutations directly.
- **Builder-hosted (custom domain or subdomain).** Framer
  (`name.framer.website`), Webflow (`name.webflow.io`), Notion-hosted sites,
  Cargo (`cargo.site`), Format (`name.format.com`), Squarespace, Super.so,
  Carrd (`name.carrd.co`), Read.cv / Posts (`read.cv/name`, `posts.cv/name`).
- **How to find:** search `"Full Name" designer portfolio`,
  `"Full Name" UX portfolio`, `"Full Name" + current employer`. Check the
  LinkedIn Contact Info and Featured sections first — designers almost always
  link their portfolio there.
- **Disambiguation:** the portfolio's About page should name the same employer,
  city, or discipline as LinkedIn. The footer or contact page usually links to
  other profiles — harvest those.

## Design Communities

- **Dribbble** — `dribbble.com/handle`. Staple for product/visual designers.
- **Behance** — `behance.net/handle`. Adobe-linked; staple for visual, brand,
  and motion designers.
- **Figma Community** — `figma.com/@handle`. Profiles publishing files,
  plugins, or widgets. Strong signal for design-systems and tooling-minded
  designers.
- **Layers** — `layers.to/handle`. Designer community and portfolios.
- **Awwwards** — designer/agency profiles for web and interaction designers.
- **Savee / Cosmos** — visual curation; weaker identity signal, useful as
  supporting evidence.
- **ADPList** — `adplist.org/mentors/name`. Mentoring profiles; signals
  community involvement and seniority.
- **How to find:** search `"Full Name" dribbble`, `"Full Name" behance`, or
  search the platform directly by name. Then check handle reuse across all
  three.
- **Disambiguation:** match the bio's stated role/employer, the linked
  portfolio, and the profile photo against LinkedIn.

## Writing & Publishing

- **Medium** — `medium.com/@handle`. Search `site:medium.com "Full Name"`.
- **Substack** — `name.substack.com`. Newsletters on design and craft.
- **Personal blog** — often a `/blog` or `/writing` path on the portfolio
  domain, or a separate domain.
- **Hashnode / Mirror / Dev.to** — for designers who also write technical or
  web3 content.
- **Speaker Deck / SlideShare** — talk decks; pair with conference appearances.
- **Disambiguation:** the author bio should cross-reference the portfolio,
  LinkedIn, or a known handle. Article topics should match the candidate's
  discipline.

## Social

- **X / Twitter** — `x.com/handle` (also `twitter.com/handle`). Bio usually
  links the portfolio.
- **Bluesky** — `bsky.app/profile/handle`.
- **Threads** — `threads.net/@handle`.
- **Mastodon** — `instance.tld/@handle` (e.g. `mastodon.design`).
- **Instagram** — relevant for visual, brand, illustration, and type designers
  who post work; less relevant for product/UX.
- **YouTube / Vimeo** — Vimeo is a staple for motion designers; YouTube for
  designers who publish process videos or talks.
- **Disambiguation:** social handles are noisy — require a bio cross-link, a
  matching photo, or a reused handle before rating above Unverified.

## Code

- **GitHub** — `github.com/handle`. Relevant for design-systems engineers,
  design-technologists, and prototypers. Check pinned repos and the profile
  README.
- **GitLab** — `gitlab.com/handle`. Less common; check if GitHub is empty.
- **CodePen** — `codepen.io/handle`. Front-of-the-front-end prototypers and
  interaction designers.
- **Disambiguation:** the GitHub profile README or website field often links
  the portfolio directly.

## Professional & Other

- **Polywork** — `polywork.com/handle`. Activity timeline across roles.
- **Wellfound (formerly AngelList)** — startup-focused profile.
- **Contra** — `contra.com/handle`. Independent / freelance designers.
- **Conference speaker pages** — search `"Full Name" + speaker` or
  `"Full Name" + conference name`. Strong seniority signal.
- **Podcast appearances** — search `"Full Name" + podcast`. Useful supporting
  evidence; rarely a primary link.
- **Disambiguation:** these pages usually state employer and role explicitly —
  easy to confirm against LinkedIn.

## Coverage note

Record every platform you check, including the ones that return nothing, so the
`Coverage Summary` section of the output reflects real coverage. "Checked,
nothing found" is a useful result — it tells the hiring manager the candidate
has no public presence there rather than leaving them to wonder.
